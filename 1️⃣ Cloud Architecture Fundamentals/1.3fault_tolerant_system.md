# 🌍 What is Fault Tolerant System

## 🟢 पहले हिन्दी में आसान समझ

**Fault Tolerant System (दोष सहन करने वाला सिस्टम)** ऐसा सिस्टम होता है जो **किसी component के fail होने पर भी बिना रुके काम करता रहता है**।

मतलब अगर:

- Server crash हो जाए
- Database node down हो जाए
- Network path fail हो जाए

तो भी application **automatic दूसरे healthy component से चलती रहती है**।

### Real Life Example

मान लो **Hospital ICU system**।

अगर एक:

- Power supply
- Oxygen line
- Monitoring device

fail हो जाए तो क्या होगा?

ICU में हमेशा:

✔ Backup generator  
✔ Multiple oxygen supply  
✔ Multiple monitoring systems  

रहते हैं।

ताकि **एक fail हो जाए तो दूसरा तुरंत काम करे**।

इसी concept को cloud में कहते हैं:

**Fault Tolerance**

---

### Fault Tolerant vs High Availability

| Feature | High Availability | Fault Tolerance |
|-------|------------------|----------------|
| Failure handling | Recover after failure | Continue without interruption |
| Downtime | Few seconds possible | Near zero downtime |
| Complexity | Medium | High |
| Cost | Moderate | Expensive |

---

## 🔵 English Explanation

A **Fault Tolerant System** is a system designed to continue operating without interruption even when one or more components fail.

Fault tolerance is achieved by introducing:

- Redundant infrastructure
- Automatic failover
- Replicated databases
- Distributed services

Unlike high availability systems, **fault tolerant systems do not require recovery time**.

They ensure **continuous operation without downtime**.

Example industries:

- Banking systems
- Air traffic control
- Healthcare systems
- Stock trading platforms

---

# 🧠 Why Fault Tolerant Systems are Important in Cloud Architecture

### Role in Production Systems

Production applications must remain available during:

- Server crashes
- Network outages
- Hardware failures
- Software bugs

Fault tolerance ensures **business continuity**.

---

### Scalability Benefits

Fault tolerant systems allow **horizontal scaling**:

Example:

```
1 server → 10 servers → 100 servers
```

Traffic automatically distributes across servers.

---

### Security Benefits

Fault tolerant architectures improve resilience against:

- **DDoS (Distributed Denial of Service)**
- Infrastructure attacks
- Service outages

---

### Cost Optimization Impact

Fault tolerance costs more infrastructure but prevents **huge downtime losses**.

Example downtime impact:

| Industry | Downtime Cost |
|--------|--------------|
| Banking | $1M/hour |
| E-commerce | $300k/hour |
| SaaS | $100k/hour |

Investing in redundancy is cheaper than downtime.

### Diagram Explanation

1️⃣ User request enters DNS  
2️⃣ CDN delivers cached content  
3️⃣ WAF filters malicious traffic  
4️⃣ Global Load Balancer distributes requests  
5️⃣ Region A handles traffic normally  
6️⃣ If Region A fails → traffic shifts to Region B  
7️⃣ Database replication ensures data consistency  
8️⃣ Object storage stores static assets

Result:

✔ Zero downtime  
✔ Automatic failover

# ⚙ How It Works (Step by Step)

Fault tolerant workflow:

```
User → DNS → CDN → WAF → Global Load Balancer → Multi Region Apps → Cache → Database Cluster → Storage
```

Step explanation:

1. User sends request to application  
2. DNS resolves domain name  
3. CDN delivers static content  
4. WAF protects application  
5. Load balancer distributes traffic  
6. Multiple application servers process request  
7. Cache stores frequently used data  
8. Database cluster replicates data  
9. Storage keeps backups and files  

If any layer fails → system continues working using redundant components.

---

## 🧱 Code Example

```hcl
# AWS provider configure kar rahe hai
provider "aws" {
  region = "ap-south-1" # Mumbai region use kar rahe hai
}

# VPC create kar rahe hai jisme network resources rahenge
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16" # private network range define kar rahe hai
}

# Application Load Balancer create kar rahe hai jo traffic distribute karega
resource "aws_lb" "app_lb" {
  name               = "fault-tolerant-alb" # load balancer ka naam
  load_balancer_type = "application" # application load balancer type
  internal           = false # internet facing load balancer
}

# Auto Scaling Group create kar rahe hai jo automatically servers add karega
resource "aws_autoscaling_group" "app_asg" {
  min_size = 2 # minimum 2 server hamesha running rahenge
  max_size = 10 # maximum 10 server auto scale ho sakte hai
}
```

# 🎯 Interview Explanation

**Interview Question**

Explain Fault Tolerant System in Cloud Architecture.

**Strong Answer**

A fault tolerant system is designed to continue operating without interruption even when one or more components fail. This is achieved through redundancy, multi-region deployment, replicated databases, and automated failover mechanisms. Cloud platforms like AWS and Azure enable fault tolerance using load balancers, auto scaling groups, multi-availability zone deployments, and distributed storage systems.

# 🧠 Cloud Architect Mindset

Cloud architects always assume **failure is inevitable**.

They ask:

- What happens if a server crashes?
- What if an entire availability zone fails?
- What if traffic suddenly spikes?

Architects design systems with:

✔ Redundancy  
✔ Automation  
✔ Monitoring  
✔ Disaster recovery  

So the system survives **any failure scenario**.
